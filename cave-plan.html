🪨 Cave Plan

Concept
🪨
Gate Room — Between Every Level
After clearing each forest level (1–9), the ninja enters a cave gate room. It does not count as a level. There is no fighting — just a toll checkpoint. Pay enough gold across runs to open the door and advance.
🤑
Money Monster — Bump to Pay
Walking into the money monster contributes 10g at a time from the ninja's current run gold toward the gate goal. Bump repeatedly to drain your gold into the gate. Once the goal is fully funded the door unlocks.
🕳️
Hole — Only Way Out (by dying)
If the gate isn't fully paid, the only exit is the hole. Stepping into it kills the ninja and ends the run. Contributions made this visit are saved — progress toward the gate persists between runs.
📊
One Gate at a Time
Only the current gate's contribution total needs to be tracked. Gates unlock sequentially — you can't skip ahead. Once a gate is paid it stays open forever; no re-payment on future runs.
Run Flow
1. Enter cave Spawn at bottom-middle with current run gold
2. Bump 🤑 monster Each bump: −10g from run, +10g toward gate goal
3a. Gate fully funded 🤑 Monster dies — path to 🚪 door is clear; walk through
3b. Gold exhausted / give up 🕳️ Step into hole — die, progress saved
Next run Gate contribution total carries over; keep chipping away
Room Layout — 9 × 9
🚪 Exit door (blocked by monster) 🔥 Impassable fire — flanking door 🤑 Monster — dies when paid 🕳️ Hole — exit by dying 🥷 Player spawn
Gate Prices — formula: cumulative max gold × 5, rounded up to nearest 10
Cave After Lvl Before Lvl Max Cum. Gold Gate Price Min Runs

Max per level: 17g pickups + 25g snake kills = 42g. Cumulative max at level N = N × 42g.
Gate price: cumulative max × 5, rounded up to nearest 10.
Min runs to afford the gate = ceil(gate price / 42g per level-1 run if farming level 1).

Implementation
Add caveGoldContributed and currentGate (1–9) to save state — both persist across runs, survive death and new game. currentGate identifies which gate the contribution belongs to so it can't be misapplied on a future run
After each forest level (1–9), transition to cave room instead of next level directly — fade to black, then fade in to cave
Walking through cave door to next forest level — fade to black, then fade in to forest. Player position carries over from their forest exit position (same existing mechanic); persist that position through the cave so it's still available when the next forest level loads
Render cave: 9×9, cave-wall border, entrance (4,8), door (4,0), fire walls (3,0)+(5,0), monster (4,1), hole (4,4)
Monster bump: contribute min(runGold, 10, remainingNeeded) per bump — capped so you never overpay, and the last bump can be less than 10g if that's all you have or all that's left
When caveGoldContributed reaches gate price: monster dies (removed from grid), path to door is clear
Walking through door: advance to next forest level; reset caveGoldContributed to 0 for next gate
Hole interaction: kills player (same as existing death flow), contributions already saved
If re-entering a cave where gate was already paid this run (continue save): render monster as dead, door walkable
When a save exists (game in progress), show only Continue on the main menu — hide New Game so the player can't accidentally discard their run
Delete New Game+ entirely — remove the NG+ game mode, all directly connected features (MOAI tile, house/house key tiles, NG+ snake variant), and any NG+ sound effects and save state fields
On bump with gold: play kaChing sound; float 💰 notify above the ninja (player tile)
On bump with 0 gold: play "nuh-uh" sound; float 💲 notify above the money monster tile
While in cave, swap inventory bar to 🚪{level} 🤑 {contributed}/{price}g; stats bar unchanged — still shows 💰{runGold} ⏺️{moves} 🕥{timer} (timer keeps running). Level shown is the one just completed — does not increment until the player walks through the door
